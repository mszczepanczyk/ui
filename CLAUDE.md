# @mariusz.sh/ui - Development Guide

## Overview

A TypeScript UI component library built on Park UI, Ark UI, and Panda CSS with a factory pattern for extensible components.

## Package Manager

**Only use npm.** This project enforces npm via `engines` in package.json.

## Adding Dependencies

**Never add dependencies manually to package.json.** Always use:

```bash
npm install <package>        # for dependencies
npm install -D <package>     # for devDependencies
```

This ensures the latest version is installed.

## Scaffolding Tools

When adding new tooling, always use official CLI generators:

- Vite: `npm create vite@latest`
- Panda CSS: `npx panda init`
- Biome: `npx @biomejs/biome init`
- Park UI: `npx @park-ui/cli init`
- React Cosmos: `npx cosmos init`

Customize the generated configs as needed.

## Component Factory Pattern

Every component uses a `create<Component>` factory. The library's default components are created using this same factory:

```tsx
// Library creates default Button
export const Button = createButton({ /* default config */ });

// Users extend with same API
import { createButton } from '@mariusz.sh/ui';
export const Button = createButton({
  variants: {
    variant: { gradient: { /* styles */ } },
    size: { xl: { /* styles */ } }
  }
});
```

## Scripts

- `npm run dev` - Start Vite dev server
- `npm run build` - Build library
- `npm run cosmos` - Start React Cosmos playground
- `npm run lint` - Run Biome linter
- `npm run format` - Format with Biome
- `npm run check` - Run all Biome checks
- `npm run typecheck` - TypeScript type checking

## Styling

Uses Panda CSS with Park UI preset. Styles are defined using:

- **Recipes** (`cva`) for component variants
- **Semantic tokens** for theming
- **Conditions** for responsive/state styles

The `styled-system/` directory is generated by Panda CSS and included in the repo.

## Testing Components

Use React Cosmos fixtures in `src/**/*.fixture.tsx`:

```tsx
// src/components/button/button.fixture.tsx
import { Button } from './button';

export default {
  primary: <Button variant="solid">Primary</Button>,
  secondary: <Button variant="outline">Secondary</Button>,
};
```
