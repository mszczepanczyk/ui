# @mariusz.sh/ui - Development Guide

## Overview

A TypeScript UI component library built on Park UI, Ark UI, and Panda CSS.

### Tech Stack
- **Package manager:** npm (enforced via `engines`)
- **Build:** Vite
- **React:** 19+ (no backward compatibility)
- **Styling:** Panda CSS + Park UI preset
- **Linting:** Biome
- **Playground:** React Cosmos
- **Target:** SPAs only (no SSR/RSC)

## Constraints

- **Only use npm** - enforced via `engines` in package.json
- **Never add dependencies manually** - always use `npm install <package>`
- **Use official CLI generators** - for scaffolding configs
- **Always run `npm run format` before committing**
- **Ask questions when unsure, stop when stuck**

## Adding Dependencies

```bash
npm install <package>        # for dependencies
npm install -D <package>     # for devDependencies
```

## Adding Components

Use Park UI CLI:

```bash
npx @park-ui/cli add <component>
```

Components are added to `src/components/` (configured in `components.json`).

## Project Structure

```
src/
├── components/          # UI components (flat structure, PascalCase)
│   ├── Button.tsx
│   ├── Button.fixture.tsx  # Cosmos fixture
│   ├── Input.tsx
│   └── ...
├── theme/
│   ├── recipes/         # Panda CSS recipes
│   ├── tokens/          # Design tokens
│   ├── colors/          # Color palettes
│   └── ...
└── index.ts             # Main exports

styled-system/           # Generated by Panda CSS (in repo)
```

## Scripts

- `npm run dev` - React Cosmos playground
- `npm run build` - Build library to dist/
- `npm run cosmos` - React Cosmos playground (alias)
- `npm run lint` - Biome linter
- `npm run format` - Biome formatter
- `npm run check` - All Biome checks
- `npm run test:visual` - Run visual regression tests
- `npm run test:visual:update` - Update visual snapshots
- `npm run test:visual:report` - Open HTML diff report
- `npm run typecheck` - TypeScript check

## Styling

Uses Panda CSS with Park UI preset (indigo accent, neutral gray, sm radius):

- **Recipes** in `src/theme/recipes/` for component variants
- **Semantic tokens** for theming (light/dark)
- **styled-system/** is generated and committed to repo

After modifying `panda.config.ts` or recipes:

```bash
npx panda codegen
```

## Testing Components

### Visual Testing

Visual regression tests are integrated with React Cosmos using Playwright:

```bash
npm run test:visual         # Run tests (fails if snapshots changed)
npm run test:visual:update  # Update baseline snapshots
npm run test:visual:report  # View HTML diff report
```

Tests automatically capture screenshots of all Cosmos fixtures. When component appearance changes:
1. Tests will fail showing visual diffs
2. Open the HTML report to review side-by-side comparisons
3. If changes are intentional, update snapshots with `npm run test:visual:update`

Snapshots are stored in `tests/visual.spec.ts-snapshots/` and should be committed to git.

### Cosmos Fixtures

Use React Cosmos fixtures for interactive development:

```tsx
// src/components/Button.fixture.tsx
import { Button } from "./Button";

export default {
  solid: <Button variant="solid">Solid</Button>,
  outline: <Button variant="outline">Outline</Button>,
};
```

Run: `npm run dev`
